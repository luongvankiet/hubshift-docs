## API Communication Patterns

### RESTful APIs
Hubshift uses RESTful API design principles:

- **Base URLs**:
  - Web APIs: `/api/*`
  - Mobile APIs: `/api/frontend/*`
  - V2 APIs: `/api/v2/*`

### Authentication
- **JWT Tokens**: JSON Web Tokens for authentication
- **Web**: Tokens stored in HTTP-only cookies
- **Mobile**: Tokens in `Authorization: Bearer <token>` header
- **Token Refresh**: Automatic token refresh mechanism

### Request/Response Format
- **Content-Type**: `application/json` for most requests
- **File Uploads**: `multipart/form-data` for file uploads
- **Response Format**: Standardized JSON responses with `success`, `message`, and `data` fields

### Example API Request Pattern

```javascript
// Frontend API call example
const response = await axios.post('/api/appointment/create', {
  clientId: 'client_id',
  healthCarerId: 'hc_id',
  date: '2024-01-15',
  startTime: '09:00',
  endTime: '17:00'
}, {
  headers: {
    'Authorization': `Bearer ${token}` // Mobile
    // Web: Token automatically sent via cookie
  }
});
```

### Error Handling
- Standardized error responses
- HTTP status codes (200, 400, 401, 403, 404, 500)
- Error messages in response body
